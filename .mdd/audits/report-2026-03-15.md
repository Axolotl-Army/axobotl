### LOW (2)

**L1 — `count` returned as string from raw SQL (02-dashboard)**
`src/dashboard/app/api/v1/commands/route.ts:22` — PostgreSQL `COUNT(*)` returns a numeric string, not a number. The type declaration `count: string` is correct but any consumer doing arithmetic must parse it first. No current consumer of this endpoint does arithmetic on `count`, so this is latent rather than active, but should be documented or coerced at the source.
- File: `src/dashboard/app/api/v1/commands/route.ts`
- Fix: Cast at source: `` COUNT(*) AS count `` with a `parseInt(row.count)` transform before returning, or add a `cast(count as int)` in SQL.

**L2 — SettingsDrawer useEffect missing deps (02-dashboard)**
`src/dashboard/layouts/components/SettingsDrawer.tsx:27` — The `useEffect` that auto-enables `navFull` when `navCollapsed` is set lists only `[navCollapsed]` as deps, omitting `toggleSetting` and `navFull`. This is intentional behavior but will trigger `react-hooks/exhaustive-deps` lint warnings and creates a stale closure risk if `toggleSetting` ever becomes non-stable.
- File: `src/dashboard/layouts/components/SettingsDrawer.tsx`
- Fix: Add `eslint-disable-next-line react-hooks/exhaustive-deps` comment with explanation, OR use a ref for `toggleSetting` and include all deps properly.
