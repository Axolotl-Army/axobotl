## Findings by Severity

### HIGH (1)

**H1: leveling/page.tsx exceeds 300-line quality gate (560 lines)**
- File: `src/dashboard/app/(admin)/dashboard/plugins/leveling/page.tsx`
- The file contains toggle, notifications, XP settings, role rewards, and save logic all in one component.
- Should be split into: PluginToggle, NotificationsCard, XpSettingsCard, RoleRewardsCard sub-components.
- Effort: ~15 min

### MEDIUM (2)

**M1: Role rewards table uses raw `<Table>` instead of DataTable**
- File: `src/dashboard/app/(admin)/dashboard/plugins/leveling/page.tsx` (lines 433-469)
- Violates the "all tables must use DataTable + TablePagination" rule from MDD doc 04 and MEMORY.md.
- Should use DataTable with TanStack Table. Pagination optional (max 25 roles).
- Effort: ~10 min

**M2: MDD docs 01, 02, 06, 07 are outdated**
- Doc 01 doesn't reference plugin system integration or LevelRole model
- Doc 02 doesn't mention plugin/settings pages added later
- Doc 06 references `/dashboard/config/` which was replaced by plugin pages
- Doc 07 LevelRole spec missing `cumulative` column (added in doc 08)
- Effort: ~10 min to update all four

### LOW (1)

**L1: No component-level tests for DataTable/TablePagination**
- MDD doc 04 lists `tests/unit/dashboard/components/DataTable.test.tsx` in test_files but the file doesn't exist
- These are ported SmartAdmin components -- lower risk, but the doc references them
- Effort: ~15 min
