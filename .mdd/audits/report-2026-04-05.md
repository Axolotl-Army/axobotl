## Findings by Severity

### HIGH

**H1. `/xp add` reads level-up template from wrong source**
- **Location:** `src/bot/commands/xp.ts:117-124`
- **Description:** When `/xp add` triggers a level-up, it reads `Guild.levelUpMessage` directly from the Guild table instead of from `GuildPlugin.config.levelUpMessage` (the plugin config). Since feature 07, the dashboard saves level-up templates to plugin config, not Guild. The `messageCreate` handler correctly reads from plugin config. This means `/xp add` uses the old pre-migration template (or null) instead of whatever the admin configured in the dashboard.
- **Impact:** Level-up messages from `/xp add` use wrong/default template when admin has configured a custom one via dashboard.
- **Fix:** Read from `GuildPlugin.config` instead of `Guild.levelUpMessage`, mirroring the pattern in `messageCreate.ts`.
- **Effort:** ~5 minutes

### MEDIUM

**M1. MDD docs list test files at wrong paths**
- **Location:** `.mdd/docs/04-smarttables.md`, `.mdd/docs/05-components-v2.md`, `.mdd/docs/07-plugin-system.md`
- **Description:** Several MDD docs reference test file paths that don't match the actual codebase:
  - Doc 04: `tests/unit/dashboard/components/DataTable.test.tsx` -- does not exist
  - Doc 05: `tests/unit/leaderboard-v2.test.ts`, `tests/unit/component-builders.test.ts` -- don't exist (actual: `tests/unit/bot/commands/leaderboard.test.ts`)
  - Doc 07: `tests/unit/pluginCache.test.ts`, `tests/unit/pluginRegistry.test.ts`, `tests/unit/levelRoles.test.ts` -- wrong paths (actual: `tests/unit/bot/plugins/pluginCache.test.ts`, `tests/unit/bot/plugins/registry.test.ts`)
- **Impact:** Docs mislead about test locations; discourages trust in MDD docs.
- **Fix:** Update test_files frontmatter in affected docs.
- **Effort:** ~5 minutes

**M2. Doc 05 claims interactionCreate handles button routing -- it doesn't**
- **Location:** `.mdd/docs/05-components-v2.md`, "Button Interaction Handling" section
- **Description:** The doc says "The interactionCreate handler must be updated to handle isButton()" and describes routing via custom ID parsing. In reality, the leaderboard uses a `createMessageComponentCollector` directly on the reply message. `interactionCreate.ts` only handles `isChatInputCommand()`. The implementation is correct, but the doc is misleading.
- **Impact:** Documentation inaccuracy -- someone reading the doc would look for button handling in the wrong place.
- **Fix:** Update doc to describe the collector pattern used.
- **Effort:** ~5 minutes

### LOW

**L1. Leaderboard member fetching is sequential**
- **Location:** `src/bot/commands/leaderboard.ts:47-57`
- **Description:** `buildLeaderboardLines` fetches members one at a time in a loop. Could batch-fetch for 10 members at once to reduce API calls.
- **Impact:** Minor perf issue -- only 10 fetches per page, and cache hits prevent most API calls.
- **Fix:** Use batch fetch or pre-fetch all members.
- **Effort:** ~10 minutes

**L2. Settings page duplicates leveling formula**
- **Location:** `src/dashboard/app/(admin)/dashboard/settings/page.tsx:26-40`
- **Description:** The settings page re-implements `getXpForLevel` and `getLevelFromXp` for the embed preview, duplicating logic from `src/bot/utils/levelUtils.ts`. If the formula changes, the preview will diverge.
- **Impact:** Cosmetic only -- preview accuracy, not data correctness.
- **Fix:** Extract shared constants/formula to a shared module.
- **Effort:** ~15 minutes

**L3. /info command not documented in any MDD feature doc**
- **Location:** `src/bot/commands/info.ts`
- **Description:** The `/info` command exists in the codebase and is registered as a base command, but no MDD feature doc covers it.
- **Impact:** Gap in documentation coverage.
- **Fix:** Add to doc 05 or create a dedicated doc.
- **Effort:** ~5 minutes

**L4. Rate limit memory cleanup is request-driven**
- **Location:** `src/dashboard/middleware.ts:30-41`
- **Description:** Stale rate-limit entries are only cleaned up when a new request arrives. If traffic stops, memory isn't freed.
- **Impact:** Negligible for typical usage patterns.
- **Fix:** None needed -- acceptable trade-off for Edge middleware.

**L5. Doc 04 test file does not exist**
- **Location:** `.mdd/docs/04-smarttables.md` test_files
- **Description:** `tests/unit/dashboard/components/DataTable.test.tsx` is documented but does not exist.
- **Impact:** Missing test coverage for table components.
- **Fix:** Create the test file or remove from doc.
- **Effort:** ~20 minutes (to create) or ~1 minute (to remove)
