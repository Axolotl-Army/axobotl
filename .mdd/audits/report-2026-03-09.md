**L1: Dead `page` variable in route handlers**
- **File:** `src/dashboard/routes/index.ts` (lines 51, 71, 91)
- **Issue:** Routes pass `page: 'home'`/`'commands'`/`'logs'` to templates, but no template uses this variable. The SmartAdmin sidebar detects active state via JavaScript URL matching. The MDD doc states the variable should be renamed to `activePage`, but it's actually unused.
- **Impact:** Dead code only. No functional impact.
- **Fix:** Remove `page` property from all three `res.render()` calls. Update MDD doc to reflect that sidebar uses JS-based active detection.
- **Effort:** 2 minutes
