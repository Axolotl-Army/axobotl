**Findings:**

- MEDIUM: app-footer.ejs line 4 uses `document.write(new Date().getFullYear())` -- anti-pattern that fails with async loading and triggers browser warnings. Should use EJS server-side rendering: `<%= new Date().getFullYear() %>`

- MEDIUM: logs.ejs lines 74-78 renders ALL pagination links in a loop (`for i=1..pagination.total`). With many pages (e.g. 500+), this creates hundreds of DOM elements. Should implement smart pagination with prev/next and ellipsis.

- LOW: app-header.ejs lines 57-60 and 68-71 repeat Discord avatar URL construction logic (DRY violation). Template-level duplication, 2 instances. Could extract to a server-side helper but impact is minimal.

- LOW: 42 unused SmartAdmin page scripts in `public/scripts/pages/`. Only `auth-animation.js` is loaded by any template. The other 42 files are dead code bloat (~680KB total).

- LOW: app-settings.ejs lines 122-123 use inline `onclick=` handlers (`appDOM.resetStyle()`, `resetPanelState()`). These are SmartAdmin vendor patterns tied to smartApp.js globals. Not a security risk (no user input), but inconsistent with `data-action` pattern used elsewhere. Classified LOW because this is vendor code.

- LOW: app-settings.ejs uses both `data-codependence` and `data-dependency` attributes (lines 35, 50). This is SmartAdmin's intended API (not a typo) -- codependence means "disable when other is active", dependency means "requires other to be active".

- INFO: No CSRF middleware in the project. Logout form (app-header.ejs line 98) uses POST without a CSRF token. Acceptable for OAuth-only auth (no password forms), but worth noting for future forms.

- INFO: Old partials (head.ejs, navbar.ejs, sidebar.ejs, scripts.ejs) are fully deleted. Clean migration.

- INFO: All EJS templates use `<%=` for user data (auto-escaping). No XSS vulnerabilities found.

**Test coverage:** 41/41 tests passing (no SmartAdmin-specific tests -- UI-only migration)
**Doc accuracy:** MDD doc matches implementation. All partials, theme config, and layout structure match as documented.
