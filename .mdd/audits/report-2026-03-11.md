## Findings by Severity

### HIGH (1)

**H1 — No authorization on dashboard (02-nextjs)**
Any Discord user who completes OAuth can access all dashboard pages and API routes. The `BOT_OWNER_ID` env var is defined in `.env.example` but never checked. The dashboard should restrict access to the bot owner.
- Files: `src/dashboard/lib/auth.ts`, all API route files
- Fix: Add `discordId` check in NextAuth `signIn` callback or create middleware that compares session `discordId` against `BOT_OWNER_ID`

### MEDIUM (4)

**M1 — Theme reset references wrong element ID (02-nextjs)**
`useLayoutContext.tsx:92` uses `document.getElementById('theme-style')` but the element is `id="app-theme"` in `app/layout.tsx:33`. The Factory Reset button doesn't clear the active theme CSS.
- Files: `src/dashboard/context/useLayoutContext.tsx`
- Fix: Change `'theme-style'` to `'app-theme'`

**M2 — Rate limiting documented but not implemented (02-nextjs)**
MDD doc claims API rate limiting (60 req/min) and page rate limiting (100 req/15min) but no middleware exists.
- Files: MDD doc, all API routes
- Fix: Either implement rate limiting or remove from documentation

**M3 — requireOwner middleware documented but missing (02-nextjs)**
MDD doc references `requireOwner` middleware that doesn't exist.
- Fix: Will be addressed by H1 fix (authorization)

**M4 — SettingsDrawer approaching line limit (02-nextjs)**
`SettingsDrawer.tsx` is 257 lines, approaching the 300-line quality gate.
- Fix: Extract theme list and individual setting sections into smaller components

### LOW (4)

**L1 — MDD doc routes incomplete (02-nextjs)**
`/api/v1/commands` and `/api/v1/logs` (with pagination) not in the routes section.
- Fix: Update MDD doc

**L2 — useEffect runs on every render (02-nextjs)**
`useLayoutContext.tsx:158` — `useEffect` depends on `settings` object which is a new reference each render.
- Fix: Memoize or use individual values

**L3 — Obsolete MDD doc (02-smartadmin)**
`.mdd/docs/02-smartadmin-dashboard.md` references deleted files.
- Fix: Remove or archive the file

**L4 — Duplicate MDD doc numbering**
Both `02-smartadmin-dashboard.md` and `02-dashboard-nextjs-migration.md` use number 02.
- Fix: Remove obsolete doc (L3) resolves this
